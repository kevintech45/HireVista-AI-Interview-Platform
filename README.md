<h1 align="center">HireVista AI Interview Platform</h1>

<p align="center">
  <b>A next-generation, fully automated AI interviewing platform built with Next.js, Vapi, Groq, and Supabase.</b>
</p>

## 🚀 Overview

**HireVista** modernizes the hiring process by replacing early-round screening calls with an intelligent, conversational Voice AI. Candidates interact directly with a real-time AI Recruiter that asks tailored questions, listens to their responses, provides hints if they get stuck, and grades their performance automatically.

## ✨ Features

- 🔐 **Google Authentication**: Secure, seamless login using Supabase Auth.
- 🎯 **Dynamic Question Generation**: Automatically crafts highly specific interview questions based on the Job Role and Description using the lightning-fast **Groq API** (`llama-3.1-8b-instant`).
- 🎙️ **Live AI Voice Interviews**: Conduct completely natural, conversational voice interviews featuring incredibly low-latency speech recognition and text-to-speech powered by **Vapi.AI** (using OpenAI `alloy` voice and Deepgram transcribing).
- 📊 **Automated AI Feedback**: Appraises candidate performance instantly after the call, saving customized feedback and scoring directly to the database.
- 🗄️ **Secure Database Architecture**: Robust Supabase PostgreSQL backend ensuring proper Row Level Security (RLS). Candidates can take interviews via secure, uniquely-generated links without needing an account.

## 🛠️ Tech Stack

- **Frontend**: Next.js 15 (App Router), React, Tailwind CSS, Shadcn UI, Lucide Icons
- **Backend/Database**: Supabase (PostgreSQL, Auth, RLS Policies)
- **Voice AI**: Vapi.AI Web SDK
- **LLM/Generative AI**: Groq (Llama 3.1) and OpenAI (GPT-4)

## 💻 Running Locally

1. **Clone the repository:**
   ```bash
   git clone https://github.com/kevintech45/HireVista-AI-Interview-Platform.git
   cd HireVista-AI-Interview-Platform
   ```

2. **Install dependencies:**
   ```bash
   npm install
   ```

3. **Set up Environment Variables:**
   Create a `.env.local` file and provide your own API keys:
   ```env
   NEXT_PUBLIC_SUPABASE_URL=your_supabase_project_url
   NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
   GROQ_API_KEY=your_groq_api_key
   NEXT_PUBLIC_VAPI_PUBLIC_KEY=your_vapi_public_key
   NEXT_PUBLIC_HOST_URL=http://localhost:3000/interview/
   ```

4. **Start the development server:**
   ```bash
   npm run dev
   ```

## 🗄️ Database Setup

Run the following SQL script in your Supabase SQL Editor to instantly recreate the required tables:

```sql
CREATE TABLE public."Users" (
    "id" uuid NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
    "name" text,
    "email" text UNIQUE,
    "picture" text,
    "credits" integer DEFAULT 10,
    "created_at" timestamp with time zone DEFAULT now(),
    PRIMARY KEY ("id")
);

CREATE TABLE public."Interviews" (
    "id" bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    "interview_id" uuid UNIQUE NOT NULL,
    "jobPosition" text,
    "jobDescription" text,
    "type" text,
    "duration" text,
    "questionList" jsonb,
    "userEmail" text,
    "created_at" timestamp with time zone DEFAULT now()
);

CREATE TABLE public."interview_feedback" (
    "id" bigint GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    "userName" text,
    "userEmail" text,
    "interview_id" uuid NOT NULL,
    "feedback" jsonb,
    "recommended" boolean DEFAULT false,
    "created_at" timestamp with time zone DEFAULT now()
);

ALTER TABLE public."interview_feedback"
ADD CONSTRAINT "interview-feedback"
FOREIGN KEY ("interview_id") REFERENCES public."Interviews"("interview_id") ON DELETE CASCADE;

ALTER TABLE public."Users" ENABLE ROW LEVEL SECURITY;
ALTER TABLE public."Interviews" ENABLE ROW LEVEL SECURITY;
ALTER TABLE public."interview_feedback" ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Users can manage own profile" ON public."Users" FOR ALL USING (auth.uid() = id);
CREATE POLICY "Public can view interviews" ON public."Interviews" FOR SELECT USING (true);
CREATE POLICY "Creators can insert interviews" ON public."Interviews" FOR INSERT WITH CHECK (auth.jwt()->>'email' = "userEmail");
CREATE POLICY "Anyone can insert feedback" ON public."interview_feedback" FOR INSERT WITH CHECK (true);
CREATE POLICY "Creators can read feedback for their interviews" ON public."interview_feedback" FOR SELECT USING (EXISTS (SELECT 1 FROM public."Interviews" WHERE public."Interviews"."interview_id" = public."interview_feedback"."interview_id" AND public."Interviews"."userEmail" = auth.jwt()->>'email'));
```
